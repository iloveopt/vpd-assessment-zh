# vpd-assessment-zh

**价值主张诊断框架** — 用于诊断你的产品是否真正解决了正确客户的正确问题。

> 基于 Strategyzer 价值主张设计框架（Osterwalder, Pigneur, Bernarda, Smith）  
> 版本：1.1.0 · 作者：Eason Zhang, Mars

---

## 这个 Skill 解决什么问题

大多数产品失败，不是因为执行力差，而是因为解决了错误的问题——或者为错误的客户解决了正确的问题。

`vpd-assessment-zh` 在机制层面诊断你的价值主张：

- 你的产品是否真正解决了客户重要的 Jobs / Pains / Gains？
- 产品市场适配是真实存在的，还是主观臆断的？
- 商业模式是否足够强健，能支撑价值主张持续运转？
- 与竞品相比，你的价值主张差异化在哪里？
- 哪些关键假设需要优先验证？

这不是填表练习，而是在投入资源之前获得诚实、结构化答案的工具。

---

## 三种模式

| 模式 | 时间 | 适用场景 | 输出格式 |
|------|------|----------|----------|
| `deep` | 90–120 分钟 | 顾问 DD、完整可行性分析 | HTML 报告（Eason Consulting 配色） |
| `medium` | 45–60 分钟 | 初步项目评估、提案前诊断 | Markdown ≤500 字 + BM 评分表格 |
| `quick` | 30 分钟 | 快速方向判断、群内问答 | Markdown ≤300 字（对外交付可选 HTML）|

---

## 五个诊断步骤（deep / medium）

1. **客户档案扫描** — 识别功能型 / 社会型 / 情感型 Jobs；排列最重要的 Pains 和 Gains
2. **价值地图分析** — 映射痛点缓解器和收益创造器；识别覆盖缺口
3. **适配度诊断** — 问题-解决方案适配 / 产品-市场适配 / 商业模式适配（🟢/🟡/🔴）
3b. **竞品 VP 对比** *（仅 deep 和 medium）* — 与 2–3 个竞品的横向对比矩阵；识别差异化优势与重叠风险
4. **商业模式质量评分** — 7 个维度 × 0–10 分；找出最薄弱维度并给出具体改进建议
5. **下一步行动** *（仅 deep）* — 创新方向建议 + 测试卡（假设 → 验证方法 → 成功标准）

## 与其他框架的区别

| 框架 | 视角 | 使用时机 |
|------|------|----------|
| **vpd-assessment-zh** | 产品-客户适配 | 基础层，最先用 |
| Superpowers | 竞争护城河与战略 | VP 验证后 |
| YC 评估 | 投资者押注价值 | 战略清晰后 |

**推荐使用顺序**：`vpd-assessment` → Superpowers → YC

---

## Skill 联动

- **deep 模式完成后** → `company-financial-analysis`（VP诊断 + 财务分析 = 完整 DD）
- **quick / medium 完成后** → `consulting-report`（将诊断结论打包成客户报告）
- **并行使用** → `venture-dd`（投资者视角补充产品-客户视角）

## 质量标准

- 所有诊断结论必须基于提供的信息——禁止编造
- 适配度结论必须区分**已验证事实**和**未验证假设**
- 信息不足时明确标注「数据不足」
- deep 模式：交付前必须过 10 个特征 QC 清单
